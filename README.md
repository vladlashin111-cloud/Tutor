# 🎓 TutorSpace — Инструкция по запуску

## Что нужно установить

1. **Node.js** — скачайте с https://nodejs.org (версия 18 или новее)
2. Больше ничего! База данных SQLite создаётся автоматически.

---

## Установка и запуск

### Шаг 1 — Скачайте проект
Скопируйте папку `tutorspace` на ваш компьютер.

### Шаг 2 — Установите зависимости
Откройте терминал (командную строку) в папке проекта и выполните:
```bash
npm install
```

### Шаг 3 — Настройте конфигурацию
```bash
# Скопируйте файл настроек
cp .env.example .env
```
Откройте файл `.env` в любом текстовом редакторе и измените:
- `JWT_SECRET` — любая длинная случайная строка (важно для безопасности!)
- `EMAIL_*` — настройки почты для восстановления пароля (необязательно)

### Шаг 4 — Запустите сервер
```bash
node server.js
```

Вы увидите:
```
═══════════════════════════════════════
  🎓 TutorSpace сервер запущен!
  📡 Адрес: http://localhost:3000
  🗄️  База данных: tutorspace.db
═══════════════════════════════════════
```

### Шаг 5 — Откройте браузер
Перейдите по адресу: **http://localhost:3000**

---

## Доступ с других устройств (ученики с других ПК)

1. Узнайте IP-адрес вашего компьютера:
   - **Windows:** откройте CMD → введите `ipconfig` → найдите `IPv4-адрес`
   - **Mac/Linux:** откройте Терминал → введите `ifconfig`

2. Все устройства должны быть в **одной Wi-Fi сети**

3. Ученики открывают в браузере: `http://ВАШ_IP:3000`
   - Например: `http://192.168.1.5:3000`

## Восстановление пароля

### Вариант A: Без email (режим разработки)
Если email не настроен — код сброса пароля появляется **в консоли сервера** (в терминале, где запущен сервер). Репетитор сообщает код ученику.

### Вариант Б: С email (Gmail)
1. В Google-аккаунте включите двухэтапную аутентификацию
2. Создайте «Пароль приложения»: myaccount.google.com → Безопасность → Пароли приложений
3. Вставьте этот пароль в `.env` файл в поле `EMAIL_PASS`

## Структура файлов

```
tutorspace/
├── server.js          ← Главный файл сервера (Node.js + Express)
├── db.js              ← База данных (SQLite)
├── package.json       ← Зависимости проекта
├── .env.example       ← Шаблон настроек
├── .env               ← Ваши настройки (создаётся вручную)
├── tutorspace.db      ← База данных (создаётся автоматически)
├── uploads/           ← Папка с загруженными файлами (создаётся автоматически)
└── public/
    └── index.html     ← Веб-интерфейс приложения
```

## Технологии

| Компонент | Технология |
|---|---|
| Сервер | Node.js + Express |
| База данных | SQLite (better-sqlite3) |
| Хеширование паролей | bcrypt (cost factor 12) |
| Авторизация | JWT (JSON Web Token) |
| Загрузка файлов | Multer |
| Email | Nodemailer |
| Интерфейс | HTML5 + CSS3 + Vanilla JavaScript |

## Частые вопросы

**Q: Данные сохраняются?**  
A: Да! Файл `tutorspace.db` — это база данных. Пока он существует, все данные сохранены.

**Q: Как сделать резервную копию?**  
A: Скопируйте файл `tutorspace.db` и папку `uploads/`.

**Q: Как остановить сервер?**  
A: Нажмите `Ctrl + C` в терминале.

**Q: Порт 3000 занят?**  
A: Измените `PORT=3001` в файле `.env`.
